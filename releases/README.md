# Описания релизов

В эту папку добавляются файлы с описанием изменений по версиям для использования в релизах на GitHub.

- **Имя файла** — номер версии (тега): `v1.0.0.md`, `v1.1.0.md` и т.д.
- **Содержимое** — краткий список по пунктам, что реализовано в этой версии (в общих словах), чтобы затем вставить в описание релиза на GitHub.

## Пример (v1.0.0.md)

```markdown
## Что сделано

- Добавлена интеграция с S3-совместимым хранилищем (Minio/Selectel).
- Настроена работа с файловым хранилищем через переменные окружения.
```

При создании нового релиза: создать тег в git, добавить в эту папку файл с тем же именем и заполнить список изменений.
