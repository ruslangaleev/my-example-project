## Теги и релизы (для разработчиков)

При необходимости вручную создать тег версии:

- **Имя тега** — семантическая версия с префиксом `v`: `v<MAJOR>.<MINOR>.<PATCH>`, например `v1.0.0`, `v1.2.3`.
  - **MAJOR** — несовместимые изменения API.
  - **MINOR** — новая функциональность с сохранением совместимости.
  - **PATCH** — исправления и мелкие изменения.
- Создание аннотированного тега:  
  `git tag -a v1.0.0 -m "Release v1.0.0"`
- Отправка тега на сервер:  
  `git push origin v1.0.0`

Описание изменений для релиза на GitHub удобно готовить в файле `releases/vX.Y.Z.md` (см. папку `releases/`).
